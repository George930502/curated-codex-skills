# Domain docs

This is a single-context repository. Read root `CONTEXT.md` and relevant files
under `docs/adr/` when they exist. Their absence is not an error: the
`domain-modeling` workflow creates them only after a term or decision is
actually resolved.

Use glossary terms consistently in issues, tests, reviews, and skill text. If a
change conflicts with an accepted ADR, name the conflict instead of silently
overriding it.
