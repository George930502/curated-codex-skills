# Roadmap

## Admission criteria

A catalog addition should:

- solve a repeatable Codex workflow with a bounded purpose;
- use native Codex capabilities where available;
- define inputs, decisions, completion, and failure behavior;
- include source and license provenance;
- be testable without credentials or changes to a real user home;
- add less maintenance burden than the value it provides.

## Near term

- Collect real-world feedback on the initial prompt and domain workflows.
- Improve compatibility tests as stable Codex surfaces become documented.
- Add unrelated skills only after they satisfy the admission criteria.
- Stabilize public interfaces toward a future `v1.0.0`.

Ideas are not commitments. Track accepted work in GitHub Issues rather than
using this file as a hidden backlog.
